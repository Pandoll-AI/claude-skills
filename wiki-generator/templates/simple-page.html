System Architecture

We built this application using a microservices architecture with three main components: frontend, API gateway, and backend services.

The frontend communicates with the backend through the API gateway. See src/api/gateway.ts for routing logic and src/services/ for individual service implementations.

🔐 Authentication

We use JWT tokens for authentication. Users log in through OAuth2, receive a token, and include it in subsequent requests.

Token validation happens in src/middleware/auth.ts. Tokens expire after 24 hours and can be refreshed using the /api/refresh endpoint.
